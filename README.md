# Система аутентификации и авторизации

## Схема разграничения прав доступа

### Основные сущности:
1. **Пользователи (users)** - зарегистрированные пользователи системы
2. **Роли (roles)** - группы прав (админ, менеджер, пользователь)
3. **Объекты доступа (resources)** - что защищаем (товары, пользователи, заказы)
4. **Правила доступа (permissions)** - кто и что может делать

### Таблицы в БД:

#### roles
- id - первичный ключ
- name - название роли (admin, manager, user)
- description - описание

#### resources
- id - первичный ключ
- name - название ресурса (products, users, orders)
- description - описание

#### permissions
- id - первичный ключ
- role_id - связь с ролью
- resource_id - связь с ресурсом
- can_view - просмотр всех записей
- can_view_own - просмотр своих записей
- can_create - создание
- can_edit - редактирование всех
- can_edit_own - редактирование своих
- can_delete - удаление всех
- can_delete_own - удаление своих

### Как это работает:
1. Каждому пользователю присваивается роль
2. Для каждой роли определено, что можно делать с каждым ресурсом
3. При запросе проверяем:
   - Есть ли у пользователя право на действие
   - Если есть can_edit_own - проверяем, принадлежит ли объект пользователю

### Пример правил для ролей:

| Роль     | Ресурс   | view | view_own | create | edit | edit_own | delete | delete_own |
|----------|----------|------|----------|--------|------|----------|--------|------------|
| admin    | products | ✅   | ✅       | ✅     | ✅   | ✅       | ✅     | ✅         |
| admin    | users    | ✅   | ✅       | ✅     | ✅   | ✅       | ✅     | ✅         |
